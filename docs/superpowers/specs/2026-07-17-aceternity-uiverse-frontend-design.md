# CampusFlow 前端视觉优化设计

日期：2026-07-17  
状态：已获用户批准  
选定方向：C「克制融合」

## 目标

在不改变 CampusFlow 现有业务流程、路由、数据结构和导出逻辑的前提下，统一优化首页、上传、结果确认、事件编辑与导出页面。视觉语言融合 Aceternity UI 的空间光感和 Uiverse.io 的微交互手感，使产品更现代、更具辨识度，同时保持校园工具所需的清晰、可靠和高效。

## 范围

- 重构全局色彩、背景、排版、边框、阴影和动效变量。
- 优化全局页头、页脚、导航及四步流程指示器。
- 重构首页首屏、事件流预览、数据指标和流程卡片。
- 统一上传、结果、编辑、导出页面的容器、表单、按钮、提示、标签、表格和空状态样式。
- 保留现有页面路由、业务文案含义、状态管理、上传解析、编辑和 ICS 导出行为。
- 不引入账户系统、持久化、第三方 UI 运行时或新业务功能。

## 视觉系统

### 色彩

- 页面底色：`#06110d` 至 `#0b1712` 的深墨绿渐变。
- 主前景：`#ecfdf5`；次级文本：`#9fb8ad`。
- 品牌强调：翡翠绿 `#6ee7b7`；强交互态使用 `#a7f3d0`。
- 辅助事件色：紫 `#c4b5fd`、玫瑰 `#fda4af`、琥珀 `#fcd34d`、天青 `#67e8f9`。
- 面板：半透明深色表面，使用低对比白色描边；重要输入可使用更高明度的实色表面保证可读性。

### 背景与层次

- 全局背景使用低密度网格、局部径向光晕和轻微噪点感，不使用大型装饰图片。
- 内容面板采用玻璃质感：透明背景、细描边、柔和阴影和有限的 `backdrop-blur`。
- 聚光效果仅用于首页首屏、主要 CTA 和当前步骤，避免每张卡片同时发光。

### 排版

- 使用系统无衬线字体栈，中文优先匹配常见系统字体，避免新增网络字体依赖。
- 标题使用紧凑字距和高字重；正文保持舒适行高。
- 英文眉题使用大写、宽字距和较小字号，作为页面节奏标识。

## 组件设计

### 全局导航

- 页头保持粘性，改为深色半透明玻璃层。
- 品牌标志简化为单一发光圆环/核心图形，避免四色方块造成视觉噪声。
- 桌面端显示文字导航；移动端维持可触达的紧凑布局。
- 导航项提供清晰的悬浮和键盘焦点状态。

### 按钮

- 主按钮使用浅翡翠底、深色文字和局部光晕；悬浮轻微上移，按下缩小。
- 次按钮使用透明面板与细描边，悬浮时提升明度。
- 禁用态取消位移和光晕，并保持足够文字对比度。
- 所有动效控制在 150–250ms。

### 卡片与面板

- 统一圆角、描边、阴影和内边距层级。
- 悬浮卡片可使用轻微上移及边框增强，不使用大角度 3D 旋转。
- 数据型密集页面以可读性优先，减少背景透明度，避免内容穿透。

### 表单与上传区

- 输入框使用深色实底、浅色描边和明确的 `focus-visible` 光环。
- 上传区以大面积虚线/实线边界和中心状态表达拖拽、加载、成功、失败。
- 文件类型预设卡保持现有选择逻辑，加入选中光晕、勾选标识和按压反馈。
- 错误、警告、成功信息使用语义色边框与图标/标签，不只依赖颜色。

### 流程指示器

- 保留上传、确认、编辑、导出四步结构和现有链接。
- 当前步骤使用翡翠光带；已完成步骤降低亮度但保持可点击；未到达步骤使用低对比状态。
- 小屏幕保留数字和短标签，避免横向溢出。

## 页面方案

### 首页

- 首屏继续采用左右双栏：左侧价值主张、双 CTA、三项数据；右侧为事件流玻璃面板。
- 加入克制的网格、径向光晕和漂浮光点；事件行使用不同事件色编码。
- 下方四步流程卡形成连续视觉路径，悬浮时显示细微扫光。

### 上传页

- 保留来源预设、文件上传、文本输入、作息表和加载/错误状态。
- 将页面组织为明确的步骤标题、来源选择和主操作面板；次要说明收敛到辅助卡片。
- 拖拽和处理状态使用边框、进度光带与文案共同表达。

### 结果页

- 统计指标改为紧凑的玻璃数据卡。
- 表格表头使用稳重深色层，行悬浮和状态标签增强扫描效率。
- 移动端继续允许横向滚动，且保留可见的滚动提示或边缘渐隐。

### 编辑页

- 保持左侧事件列表、右侧编辑内容的桌面布局。
- 选中事件使用发光边框而非大面积高亮；输入区使用更高实色度保证长期编辑舒适。
- 停课日期、补充事件和导出 CTA 统一为同一设计系统。

### 导出页

- 以成功状态作为视觉中心，使用柔和的翡翠光环和 ICS 标记。
- 三种日历导入说明使用等高卡片；返回与重新开始操作保持主次清晰。

## 交互与动效

- 页面进入使用一次性的淡入与轻微上移，元素错峰不超过 80ms。
- 按钮、卡片、导航和来源选项提供悬浮、按压和键盘焦点反馈。
- 扫光只在主要 CTA 或少量流程卡使用，不持续循环干扰操作。
- 加载状态可使用低频光带或脉冲，错误状态不使用抖动。
- 在 `prefers-reduced-motion: reduce` 下关闭非必要位移、扫光和漂浮动画。

## 响应式与无障碍

- 主要断点沿用现有 Tailwind 结构；移动端优先保持单列阅读和不小于 44px 的触控目标。
- 正文、按钮、表单和状态标签满足可读对比度；低对比装饰不承载信息。
- 保留语义化标题、表格、标签和导航；交互元素使用真实按钮或链接。
- 所有可操作元素提供 `focus-visible` 状态。
- 不通过仅悬浮才能访问关键内容。

## 实现边界

- 基于现有 Next.js、React 19、Tailwind CSS 4 和少量 CSS 关键帧实现。
- 不直接复制 Aceternity UI 或 Uiverse.io 的组件代码；只借鉴视觉与交互模式。
- 优先建立少量共享 CSS 工具类和可复用 UI 组件，避免在各页面复制长串样式。
- 不修改 Zustand store、API、解析逻辑、PDF/OCR 流程或 ICS 生成逻辑。
- 若实施时发现现有乱码文本，需将其作为独立问题报告；本次视觉优化不擅自改变业务内容。

## 验证标准

- `npm run build` 成功。
- `npm run lint` 成功，或明确记录与本次变更无关的既有问题。
- `npm run test:core` 成功。
- 首页、上传、结果、编辑、导出在桌面和移动布局下无明显溢出。
- 键盘可访问所有主要操作，焦点清晰。
- 开启减少动态效果后，不出现持续移动或扫光。
- 上传、解析、编辑、选择、导出等既有交互行为不变。
