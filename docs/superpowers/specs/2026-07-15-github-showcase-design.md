# CampusFlow GitHub 展示面设计

## 目标

将 CampusFlow 的 GitHub 首页从 Next.js 默认说明改造成面向 AI 应用开发与全栈开发岗位的项目展示页。招聘方应在 30 秒内理解项目解决的问题、核心处理链路、工程难点、完成度以及如何运行项目。

## 受众与定位

- 主要受众：AI 应用开发、全栈开发相关实习与校招招聘方。
- 次要受众：希望本地运行、理解实现或继续贡献的开发者。
- 核心定位：不是通用聊天套壳，而是把校园图片、PDF 和文本可靠转换为结构化日历事件的 AI 工程项目。
- 内容语言：中文为主；首屏提供一句英文摘要，避免维护两套容易失步的完整文档。

## 展示策略

采用“招聘导向型 README”为主体，并吸收产品展示型方案的一张高质量首屏图。所有能力和数字必须能从仓库或验证结果中获得证据，不使用未经验证的用户量、准确率或性能指标。

## README 信息结构

1. **首屏 Hero**
   - 项目名称与一句话价值主张。
   - 简短英文摘要。
   - 技术栈、测试、许可证或部署状态徽章；只展示真实可验证的状态。
   - 在线体验、功能概览和快速开始锚点。
   - 一张真实产品截图；如果暂时没有合适截图，则保留明确的截图位置而不使用虚假界面图。

2. **为什么做 CampusFlow**
   - 描述课表、考试通知和作业截止时间分散在图片、PDF 与文本中的实际问题。
   - 用一句话说明从非结构化输入到日历事件的结果。

3. **核心能力**
   - 支持图片、PDF、文本等输入。
   - 支持课程、考试、作业和通知等事件类型。
   - 支持单双周、指定周次、学期起始日和停课日期。
   - 支持编辑校验与 ICS 导出。
   - 支持移动端上传与失败降级。

4. **处理链路**
   - 使用 GitHub 原生 Mermaid 图呈现：输入识别、OCR/PDF 提取、解析路由、规则或 AI 解析、校验编辑、周次计算、ICS 导出。
   - 图中明确失败路径与降级策略，突出工程可靠性。

5. **技术亮点**
   - 多来源解析路由，而非单一模型调用。
   - OCR 噪声修复和表格行重建。
   - PDF 多级解析与 serverless/mobile fallback。
   - 周次引擎、节假日排除和 ICS 生成。
   - 浏览器兼容与移动端上传策略。

6. **工程质量**
   - 展示经本地验证的 51 项核心测试全部通过。
   - 列出测试覆盖的关键领域，不承诺未测量的覆盖率。
   - 给出 `npm run test:core` 和 `npm run lint` 命令。

7. **技术栈与目录结构**
   - 只列关键依赖与用途，避免徽章墙。
   - 用精简目录树解释页面、解析器、OCR/PDF、日历与状态管理边界。

8. **本地运行与配置**
   - 给出 Node.js、安装、开发、测试和构建命令。
   - 罗列变量名，但绝不展示真实密钥或 `.env.local` 内容。
   - 解释无外部 AI Key 时仍可使用哪些本地能力。

9. **限制与路线图**
   - 诚实说明 OCR 结果受图片质量和学校模板影响。
   - 路线图只保留合理的近期方向，不承诺日期。

## 视觉规范

- 使用简洁的居中首屏，颜色与产品现有蓝绿色视觉保持一致。
- 徽章控制在一行内，不堆砌无关技术图标。
- 截图使用仓库相对路径，存放在 `docs/images/`，保证 GitHub 可直接渲染。
- Mermaid 图使用短节点文本，兼顾 GitHub 明暗主题。
- 正文标题层级不超过三级，段落保持短小，优先使用可快速扫描的列表和表格。

## 文件范围

- 必改：`README.md`。
- 可新增：`docs/images/` 下的真实截图或展示图。
- 可选：若仓库暂无许可证，则 README 不展示 License 徽章，也不在本次擅自选择许可证。
- 不改业务逻辑、API、数据库结构或生产配置。

## 验证标准

- README 中不存在 Next.js 默认模板文案、占位符、失效的仓库内相对链接或泄露的密钥。
- Mermaid 语法结构完整，Markdown 标题和代码块成对闭合。
- 所有功能描述能在源码、测试或现有部署配置中找到依据。
- `npm run test:core` 仍为 51 项通过，`npm run lint` 通过。
- 若涉及业务文件以外的展示素材，不影响 `npm run build`。

## 非目标

- 本次不重做产品 UI。
- 本次不虚构用户数据、性能结果或 OCR 准确率。
- 本次不发布新版本、不改变仓库可见性，也不自动推送到远程。
- 本次不引入仅为 README 服务的新运行时依赖。
